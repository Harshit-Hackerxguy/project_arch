# docs/

This directory contains all project-wide documentation for `project_arch`.

Documentation here describes **the project as a whole** — its architecture,
philosophy, conventions, and operational procedures. Tool-specific documentation
lives alongside the tool's configuration files.

## Documentation Philosophy

Documentation in this project is written to be **genuinely useful**, not ceremonial.

Every document answers a real question that an engineer would have when working
with this repository. If a document does not answer a real question, it should
not exist.

Documents are written for:
- A **new contributor** who has never seen this repository
- A **future maintainer** (possibly yourself) returning after months away
- A **user** performing a fresh install on a new machine

---

## Writing Standards

- Use clear, direct language — avoid jargon unless it is defined
- Keep paragraphs short
- Use tables for structured comparisons
- Use code blocks with language tags for all code samples
- Every heading should be sentence-readable without context

See [`conventions.md`](conventions.md) for the full Markdown style guide.
